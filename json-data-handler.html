Java 17 Spring Boot JSON Jackson Tutorial

Project Overview

This project demonstrates best practices for working with JSON data in a Spring Boot 17 application using the Jackson library. It serves as both a reference guide and a runnable example repository.

Whether you're building REST APIs that consume third-party JSON feeds or exposing your own endpoints, this guide walks through every common scenario you'll encounter in real-world development.

Each section includes a working code snippet, an explanation of the Jackson annotations involved, and notes on common pitfalls to avoid.

  • JSON Deserialization Map raw JSON strings and HTTP request bodies to Java POJOs using @JsonProperty and ObjectMapper.
  • JSON Serialization Convert Java objects to JSON responses with fine-grained control over field naming, nulls, and date formats.
  • Nested Objects Handle complex payloads with nested arrays, maps, and polymorphic types using @JsonTypeInfo.
  • Error Handling Catch JsonProcessingException and return structured error responses with @ControllerAdvice.

Code Examples

Deserialization — JSON → Java Object

Java 17 
// Java 17 record as the target POJO
public record UserRequest(
    @JsonProperty("first_name") String firstName,
    @JsonProperty("last_name")  String lastName,
    @JsonProperty("email")      String email
) {}

// Spring Boot controller — Jackson deserializes automatically
@RestController
@RequestMapping("/api/users")
public class UserController {

    @PostMapping
    public ResponseEntity<String> createUser(@RequestBody @Valid UserRequest req) {
        // req.firstName() / req.lastName() / req.email() ready to use
        return ResponseEntity.ok("Created: " + req.firstName());
    }
}

Serialization — Java Object → JSON

Java 17 
@JsonNaming(PropertyNamingStrategies.SnakeCaseStrategy.class)
public record UserResponse(
    Long   userId,      // → "user_id"
    String firstName,   // → "first_name"
    String email,
    @JsonIgnore String passwordHash  // excluded from output
) {}

// ObjectMapper configuration bean (global defaults)
@Configuration
public class JacksonConfig {

    @Bean
    public ObjectMapper objectMapper() {
        return new ObjectMapper()
            .registerModule(new JavaTimeModule())
            .disable(SerializationFeature.WRITE_DATES_AS_TIMESTAMPS)
            .setSerializationInclusion(JsonInclude.Include.NON_NULL);
    }
}

// Incoming JSON:
// { "order_id": 42, "items": [{"sku":"ABC","qty":2}] }

public record OrderItem(
    String sku,
    @JsonProperty("qty") int quantity
) {}

public record OrderRequest(
    @JsonProperty("order_id") Long orderId,
    List<OrderItem> items
) {}

// Manual deserialization with TypeReference for generic types
String json = """
    [{"sku":"ABC","qty":2},{"sku":"XYZ","qty":5}]
    """;

List<OrderItem> items = objectMapper.readValue(
    json, new TypeReference<List<OrderItem>>() {}
);

Error Handling with @ControllerAdvice

Java 17 
@RestControllerAdvice
public class GlobalExceptionHandler {

    @ExceptionHandler(HttpMessageNotReadableException.class)
    public ResponseEntity<ProblemDetail> handleMalformedJson(
            HttpMessageNotReadableException ex) {

        ProblemDetail problem = ProblemDetail
            .forStatusAndDetail(HttpStatus.BAD_REQUEST, "Malformed JSON body");
        problem.setTitle("Invalid Request");
        return ResponseEntity.badRequest().body(problem);
    }

    @ExceptionHandler(MethodArgumentNotValidException.class)
    public ResponseEntity<ProblemDetail> handleValidation(
            MethodArgumentNotValidException ex) {

        String detail = ex.getBindingResult().getFieldErrors()
            .stream()
            .map(e -> e.getField() + ": " + e.getDefaultMessage())
            .collect(Collectors.joining(", "));

        ProblemDetail problem = ProblemDetail
            .forStatusAndDetail(HttpStatus.UNPROCESSABLE_ENTITY, detail);
        problem.setTitle("Validation Failed");
        return ResponseEntity.unprocessableEntity().body(problem);
    }
}
